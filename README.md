
# Employee Management System

Клиент-серверное приложение для учета сотрудников предприятия.

## Описание проекта

Приложение предназначено для управления данными предприятия:

- отделами;
- должностями;
- сотрудниками.

Система позволяет выполнять CRUD-операции через графический web-интерфейс:

- просматривать список записей;
- добавлять новые записи;
- редактировать существующие записи;
- удалять записи с учетом ограничений бизнес-логики.

Проект реализован с использованием Java Servlet и JSP. Взаимодействие с базой данных выполняется через JDBC API.

## Архитектура приложения

Проект построен по классической слоистой архитектуре:

```text
Browser
   ↓
Servlet
   ↓
Service
   ↓
DAO
   ↓
DatabaseConnection
   ↓
Red Database
````

### Назначение слоев

#### Servlet layer

Servlet-слой отвечает за обработку HTTP-запросов пользователя.

В проекте реализованы следующие servlet-классы:

* `DepartmentServlet`
* `PositionServlet`
* `EmployeeServlet`

Servlet получает параметры запроса, вызывает методы Service-слоя и передает данные на JSP-страницы.

#### Service layer

Service-слой содержит бизнес-логику приложения.

В проекте реализованы следующие service-классы:

* `DepartmentService`
* `PositionService`
* `EmployeeService`

Service-слой проверяет корректность данных и контролирует выполнение бизнес-правил перед обращением к DAO.

#### DAO layer

DAO-слой отвечает за прямое взаимодействие с базой данных через JDBC.

В проекте реализованы следующие DAO-классы:

* `DepartmentDao`
* `PositionDao`
* `EmployeeDao`

DAO-классы выполняют SQL-запросы с использованием:

* `Connection`
* `PreparedStatement`
* `ResultSet`

#### Model layer

Model-классы представляют сущности предметной области:

* `Department`
* `Position`
* `Employee`

Они используются для передачи данных между слоями приложения.

#### View layer

JSP-страницы используются как слой представления.

Servlet передает данные на JSP через `request.setAttribute()`, после чего JSP отображает данные в виде HTML-страниц с использованием JSTL и Expression Language.

## Структура базы данных

База данных состоит из трех основных таблиц:

```text
departments
positions
employees
```

### departments

Таблица хранит информацию об отделах предприятия.

Поля:

* `id` — идентификатор отдела;
* `department_name` — название отдела;
* `department_phone` — рабочий телефон отдела;
* `department_email` — рабочий email отдела.

### positions

Таблица хранит информацию о должностях.

Поля:

* `id` — идентификатор должности;
* `position_name` — название должности;
* `position_salary` — зарплата по должности.

### employees

Таблица хранит информацию о сотрудниках.

Поля:

* `id` — идентификатор сотрудника;
* `full_name` — ФИО сотрудника;
* `department_id` — идентификатор отдела;
* `position_id` — идентификатор должности;
* `is_department_head` — признак того, является ли сотрудник начальником отдела.

## Связи между таблицами

```text
employees.department_id → departments.id
employees.position_id   → positions.id
```

Один отдел может содержать много сотрудников.

Одна должность может быть назначена многим сотрудникам.

Один сотрудник может быть назначен начальником отдела через поле `departments.head_id`.

## Нормализация базы данных

Структура базы данных соответствует требованиям третьей нормальной формы.

Данные разделены на отдельные сущности:

* отделы хранятся в таблице `departments`;
* должности хранятся в таблице `positions`;
* сотрудники хранятся в таблице `employees`.

Зарплата хранится в таблице должностей, так как она относится к должности, а не дублируется в таблице сотрудников.

Количество сотрудников в отделе не хранится отдельным полем, а вычисляется SQL-запросом через `COUNT()`. Это позволяет избежать дублирования данных и рассинхронизации.

## Основная функциональность

### Отделы

Для отделов реализованы следующие возможности:

* просмотр списка отделов;
* добавление нового отдела;
* редактирование отдела;
* удаление отдела;
* отображение начальника отдела;
* отображение количества сотрудников в отделе.

Удаление отдела запрещено, если в нем есть сотрудники.

### Должности

Для должностей реализованы следующие возможности:

* просмотр списка должностей;
* добавление новой должности;
* редактирование должности;
* удаление должности;
* отображение зарплаты по должности.

Удаление должности запрещено, если она назначена хотя бы одному сотруднику.

### Сотрудники

Для сотрудников реализованы следующие возможности:

* просмотр списка сотрудников;
* добавление нового сотрудника;
* редактирование сотрудника;
* удаление сотрудника;
* отображение отдела сотрудника;
* отображение должности сотрудника;
* отображение зарплаты по должности;
* отображение статуса сотрудника.
* нельзя удалить сотрудника, если он назначен начальником отдела;

При добавлении и редактировании сотрудника контролируется корректность назначения начальника отдела.

## Бизнес-правила

В приложении реализованы следующие бизнес-правила:

* нельзя удалить отдел, если в нем есть сотрудники;
* нельзя удалить должность, если она назначена сотрудникам;
* нельзя удалить сотрудника, если он назначен начальником отдела;
* в одном отделе может быть только один начальник отдела;
* начальником отдела может быть только сотрудник с должностью `Начальник отдела`;
* если у сотрудника выбрана должность `Начальник отдела`, он должен быть отмечен как начальник отдела;
* название отдела не может быть пустым;
* телефон отдела не может быть пустым;
* email отдела не может быть пустым;
* название должности не может быть пустым;
* зарплата должна быть больше 0;
* ФИО сотрудника не может быть пустым;
* сотрудник должен быть привязан к отделу;
* сотрудник должен быть привязан к должности.

## Web-интерфейс

Пользовательский интерфейс реализован с помощью JSP и Bootstrap.

Основные страницы:

```text
/                       — главная страница
/departments            — список отделов
/departments?action=new — форма добавления отдела
/positions              — список должностей
/positions?action=new   — форма добавления должности
/employees              — список сотрудников
/employees?action=new   — форма добавления сотрудника
```

Для редактирования используется параметр `action=edit`.

Например:

```text
/departments?action=edit&id=1
/positions?action=edit&id=1
/employees?action=edit&id=1
```

Операции добавления, обновления и удаления выполняются через POST-запросы.

После POST-запросов используется `sendRedirect()`, чтобы избежать повторной отправки формы при обновлении страницы.

## Настройка базы данных

Для работы приложения используется Red Database 3.0.

Перед запуском приложения необходимо создать базу данных и выполнить SQL-скрипты из папки `db`.

Порядок выполнения:

1. Создать базу данных через Red Database (или Red Expert).
2. Выполнить скрипт:

```text
db/schema.sql
```

3. Выполнить скрипт:

```text
db/test_data.sql
```

Скрипт `schema.sql` создает таблицы и связи между ними.

Скрипт `test_data.sql` добавляет тестовые данные.

## Настройка подключения к базе данных

Файл подключения находится по пути:

```text
employee-management/src/main/resources/db.properties
```

Пример содержимого:

```properties
db.url=jdbc:firebirdsql://localhost:3050/C:/path/to/company_employees.fdb
db.username=SYSDBA
db.password=your_password
```

Файл `db.properties` содержит локальный путь к базе данных и пароль, поэтому он не должен публиковаться в открытом репозитории.

Для примера можно создать файл:

```text
db.example.properties
```

и указать в нем шаблон настроек подключения.

## Сборка проекта

Для сборки проекта используется Maven.

Перейти в директорию Maven-проекта:

```bash
cd employee-management
```

Выполнить сборку:

```bash
mvn clean package
```

После успешной сборки будет создан WAR-файл:

```text
target/employee-management.war
```

Также для запуска в IntelliJ IDEA можно использовать артефакт:

```text
employee-management:war exploded
```

## Запуск приложения

Для запуска используется Apache Tomcat 10.1.

Порядок запуска:

1. Установить Apache Tomcat 10.1.
2. Открыть проект в IntelliJ IDEA.
3. Создать Tomcat run configuration.
4. В Deployment добавить:

```text
employee-management:war exploded
```

5. Указать application context:

```text

```

6. Запустить Tomcat.

После запуска приложение будет доступно по адресу:

```text
http://localhost:8080/employee-management/
```

## Особенности реализации

### JDBC

Взаимодействие с базой данных реализовано через JDBC API.

Для выполнения SQL-запросов используются `PreparedStatement`, что позволяет безопасно передавать параметры в SQL-запросы.

### JSP и JSTL

JSP используется для отображения данных пользователю.

JSTL используется для циклов и условий на JSP-страницах:

* `<c:forEach>`
* `<c:if>`
* `<c:choose>`
* `<c:when>`
* `<c:otherwise>`

### Bootstrap

Bootstrap используется для оформления интерфейса:

* таблиц;
* кнопок;
* форм;
* карточек;
* сообщений об ошибках.

### Lombok

Lombok используется в model-классах для генерации getter’ов, setter’ов и конструкторов.

Используемые аннотации:

* `@Getter`
* `@Setter`
* `@NoArgsConstructor`
* `@AllArgsConstructor`
